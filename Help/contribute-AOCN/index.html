如何给AOCN做贡献？

由于资金原因，本站使用Hexo静态部署在Coding上，因此也无法提供在线编辑的后台。如果懒得进行下面这些操作可以直接联系站长，但考虑到站长常年不在线仍然建议自己进行操作。现在进行这些操作略显繁琐，等有空了我做一个GUI管理工具进行操作。下面以Windows为例讲一下如何进行操作。

建议有一点点代码基础的人使用，有任何疑问随时联系站长

Step 0：安装必要软件

需要安装以下软件：

  1. git
  2. node.js (npm)
  3. hexo

首先安装git

然后安装node.js，同时会自动安装npm

接下来打开PowerShell（macOS下是终端），检查一下安装是否成功。输入以下指令：

# 检查git是否安装成功
git --version
# 输出：git version 2.x.x.windows.x

# 检查nodejs是否安装成功
node --version
# 输出：v14.x.x

# 检查npm是否安装成功
npm --version
# 输出：6.x.x

如图：

然后输入指令：

npm install --global hexo-cli --registry=https://registry.npm.taobao.org

执行完成后重启命令行，输入hexo --version检查是否安装成功：

除此以外，你还需要一个好用的Markdown编辑器和一个图床，这里建议使用Typora编辑器和PicGo图床。

然后进行必要配置。在命令行里输入：

git config --global user.name "<username>"
git config --global user.email "<email>"

其中<username>替换为你的名字，<email>替换为你的电子邮箱

Step 1：克隆源码仓库

在一个文件夹里打开命令行，输入以下指令：

git clone https://e.coding.net/listener1379/ao/AOCN-source.git

这时当前目录下会出现一个叫做AOCN-source的新文件夹。进入这个文件夹，输入：

npm install --force

Step 2：开始写作

命令行输入：

hexo new <new-post>

其中<new-post>为新文章的文件名，建议使用全英文并且不添加空格。

然后进入source/_post目录，其中会出现一个文件名为<new-post>.md的文件，用typora打开。同时可以在AOCN-source目录下输入hexo s指令，然后再在浏览器里打开localhost:4000实时预览

编辑完成后在命令行输入Ctrl+C关闭实时预览，在AOCN-source目录下输入：

hexo g
hexo d
# 或者可以缩写为一个指令：hexo g -d

AOCN-source目录下会出现一个叫做public的文件夹。这就是生成的静态网站文件，这些文件已经同步到了Coding上的仓库中进行自动部署。注意，自动部署完成与本机发出部署命令会有一定的延迟。

Step 3：同步源码文件

仍然是在AOCN-source目录下打开命令行，输入：

git add .
git commit -m "<commit>"
git push origin master

其中<commit>改为你对你所做修改的描述。
